# CI/CD Pipeline для Tauri приложения

Этот проект настроен с автоматическим CI/CD pipeline для сборки и распространения приложения на всех поддерживаемых платформах.

## Обзор

Pipeline автоматически:
- ✅ Проверяет изменения в коде
- ✅ Выполняет `cargo check` для проверки Rust кода
- ✅ Собирает приложение для всех платформ
- ✅ Создает релизы и pre-release
- ✅ Кеширует зависимости для ускорения сборки

## Структура Workflow

### 1. Проверка изменений (`check-changes`)
- Анализирует изменения в последнем коммите
- Проверяет изменения в `src/` и `src-tauri/` директориях
- Проверяет изменения в конфигурационных файлах
- Пропускает сборку если изменения не касаются кода

### 2. Проверка Rust кода (`cargo-check`)
- Выполняет `cargo check` для проверки синтаксиса и типов
- Останавливает сборку при наличии ошибок

### 3. Сборка для платформ
- **Linux**: AppImage, DEB, RPM пакеты
- **Windows**: MSI установщик, NSIS setup
- **macOS**: DMG установщик, App bundle (Intel & Apple Silicon)
- **Android**: APK и AAB файлы

### 4. Создание релизов
- **main ветка**: Создает стабильный релиз
- **dev ветка**: Создает pre-release с номером сборки

## Поддерживаемые платформы

### Linux
- **AppImage**: Портативный исполняемый файл
- **DEB**: Пакет для Debian/Ubuntu
- **RPM**: Пакет для Red Hat/Fedora

### Windows
- **MSI**: Windows Installer пакет
- **NSIS**: Setup.exe установщик

### macOS
- **DMG**: Disk Image для установки
- **App Bundle**: .app пакет
- Поддержка Intel (x86_64) и Apple Silicon (aarch64)

### Android
- **APK**: Android Package для установки
- **AAB**: Android App Bundle для Google Play Store

## Кеширование

Pipeline использует многоуровневое кеширование для ускорения сборки:

1. **Rust зависимости**: `~/.cargo/registry`, `~/.cargo/git`, `target`
2. **npm зависимости**: `~/.npm`
3. **Tauri build**: `src-tauri/target`, `dist`
4. **Android build**: `src-tauri/gen/android`, `dist`

Кеш автоматически инвалидируется при изменении:
- `Cargo.lock`
- `package-lock.json`
- `tauri.conf.json`

## Условная сборка

Pipeline автоматически определяет, нужно ли собирать приложение:

### Сборка выполняется при изменениях в:
- `src/` - исходный код фронтенда
- `src-tauri/` - исходный код Rust/Tauri
- `package.json` - зависимости Node.js
- `Cargo.toml` - зависимости Rust
- `tauri.conf.json` - конфигурация Tauri
- `vite.config.ts` - конфигурация сборки
- `tailwind.config.ts` - конфигурация стилей
- `tsconfig.json` - конфигурация TypeScript

### Сборка пропускается при изменениях в:
- `README.md`
- `.gitignore`
- Документация
- Другие файлы, не влияющие на сборку

## Настройка

### 1. Создание веток
```bash
# Создайте dev ветку для разработки
git checkout -b dev

# Основная разработка в dev ветке
# Слияние в main только для релизов
```

### 2. Настройка версий
Обновляйте версию в `package.json`:
```json
{
  "version": "1.0.0"
}
```

### 3. Настройка подписи (опционально)
Для подписи приложений добавьте секреты в GitHub:
- `WINDOWS_CERTIFICATE` - сертификат для Windows
- `MACOS_CERTIFICATE` - сертификат для macOS
- `ANDROID_KEYSTORE` - keystore для Android

## Использование

### Разработка
1. Работайте в `dev` ветке
2. При каждом push создается pre-release
3. Тестируйте pre-release перед слиянием в main

### Релиз
1. Слейте изменения в `main` ветку
2. Автоматически создается стабильный релиз
3. Все артефакты прикрепляются к релизу

### Мониторинг
- Проверяйте статус сборки в GitHub Actions
- Артефакты доступны в разделе Actions
- Релизы создаются автоматически

## Troubleshooting

### Ошибки сборки
1. Проверьте логи `cargo-check` job
2. Убедитесь, что все зависимости установлены
3. Проверьте конфигурацию в `tauri.conf.json`

### Проблемы с кешем
1. Кеш автоматически сбрасывается при изменении зависимостей
2. Принудительный сброс: измените версию в `package.json`

### Проблемы с Android
1. Убедитесь, что Android SDK настроен правильно
2. Проверьте `minSdkVersion` в `tauri.conf.json`
3. Убедитесь, что все Android зависимости установлены

## Оптимизация

### Ускорение сборки
- Используйте кеширование (уже настроено)
- Разбивайте большие изменения на мелкие коммиты
- Используйте `dev` ветку для тестирования

### Сокращение времени
- Сборка выполняется только при необходимости
- Параллельная сборка для разных платформ
- Кеширование между сборками

## Безопасность

- Все секреты хранятся в GitHub Secrets
- Сборка выполняется в изолированных контейнерах
- Артефакты автоматически удаляются через 30 дней

## Поддержка

При возникновении проблем:
1. Проверьте логи GitHub Actions
2. Убедитесь, что все зависимости актуальны
3. Проверьте конфигурацию Tauri
4. Создайте issue в репозитории
